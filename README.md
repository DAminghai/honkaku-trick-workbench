# 诡计拆解工作台

> 给本格推理读者与创作者的本地工具：把读过的书拆成可复用的写作素材，连接跨书的诡计异同，并从"没人组合过的诡计"里找原创灵感。

纯前端、零安装、无后端、无账号。所有数据只存在你自己的浏览器里。

---

## 这是什么

读了很多本格 / 新本格，脑子里的诡计、线索、反转却散落各处，很难复用——这个工具帮你把它们结构化下来：

- **读** 一本书，**拆** 解它的诡计；
- 把书与书之间的手法异同**连**起来；
- 在技法与题材的交叉里，找到还没人写过的组合，作为原创**灵感**。

适合：写本格推理的作者、想系统性精读诡计的读者、做类型文学研究 / 笔记的人。

## 功能

- **书单** —— 一份按构造技法分区的本格 / 新本格 / 黄金年代书单（约 70 本）作为起点，可自由勾选已读、增删书目、打题材标签，按分区 / 标签 / 作者三种视图浏览。分区可自定义新建、重命名、删除。
- **拆解库** —— 为每本书记录诡计核心、线索布置、反转点、公平性自评、可借鉴点。技法标签可自定义扩展。
- **单本关系图** —— 用节点-连线画出一本书内部的人物 / 线索 / 诡计关系（侦探板）。
- **关系网** —— 把书做成节点，建立带类型和异同备忘的跨书关联；可在图上直接拖连、点连线编辑备注，也可用表单录入。
- **灵感矩阵** —— 技法 × 题材 的交叉表，空格代表"还没人组合过的诡计"，点击即生成一张灵感卡。
- **偏食分析** —— 标签 / 作者分布与技法盲区，照见自己的阅读与写作取向。
- **灵感箱** —— 随手记原创诡计点子，挂到启发它的书上。
- **对照** —— 并排比较两本书的拆解，一键存为关系网关联。
- **导出** —— 一键导出 / 导入 JSON 备份，导出 Markdown 素材库，或把矩阵与关系图导出为 PNG。

## 快速开始

**方式一 · 直接打开**

下载仓库（Code → Download ZIP），解压后双击 `index.html`，浏览器即可使用。

**方式二 · 本地服务器**（存储与图谱加载更稳定）

```bash
# 在文件所在目录运行
python -m http.server 8000
# 然后访问 http://localhost:8000/
```

**方式三 · 在线访问**

如果本仓库开启了 GitHub Pages，可直接在浏览器打开对应网址，无需下载。

> 「单本关系图」与「关系网」会从 CDN 加载图形库 [vis-network](https://visjs.github.io/vis-network/)，首次使用需联网，之后由浏览器缓存。其余功能完全离线可用。

## 文件结构

| 文件 | 作用 |
|------|------|
| `index.html` | 应用本体（界面 + 逻辑） |
| `books.js` | 内置书单数据，与代码分离 |
| `README.md` | 本说明 |
| `LICENSE` | MIT 许可 |

`books.js` 通过 `<script>` 标签随 `index.html` 自动加载，两个文件放在同一目录即可（`file://` 本地双击亦可）。

## 自带 / 修改书单

内置书单只是一个起点，你可以在应用内随意增删，也可以直接编辑 `books.js`。每条格式：

```js
[分区号, "书名", "作者", "一句话概括", 是否默认已读(1 或 0)]
```

分区号 0–9 的含义见 `books.js` 顶部注释。欢迎通过 Pull Request 贡献优质书目与点评。

## 数据与隐私

- 没有账号、没有服务器、没有任何追踪。所有数据保存在你浏览器的 localStorage 中。
- localStorage 可能被浏览器清理，请在「数据」页定期**导出 JSON 备份**；更换设备或文件版本时，导出旧数据再导入即可迁移。

## 技术

原生 HTML / CSS / JavaScript 单页应用，无构建步骤、无框架；图谱可视化使用 vis-network。
