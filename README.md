# 本格诡计拆解工作台

一个给本格推理读者 / 创作者的本地工具：把读过的书拆成可复用的写作素材，连接跨书的诡计异同，并从"没人组合过的诡计"里找原创灵感。

零后端、零安装、纯前端。所有数据存在你自己的浏览器本地。

## 文件结构

| 文件 | 作用 |
|------|------|
| `index.html` | 应用本体（逻辑 + 界面） |
| `books.js` | 内置书单数据，与代码分离，增删 / 协作只改这个文件 |
| `README.md` | 本说明 |
| `LICENSE` | MIT |

双击 `index.html` 即可用浏览器打开（`books.js` 通过 `<script>` 标签同目录加载，本地 `file://` 也能用）。

## 功能

- **书单**：按构造技法分区的本格 / 新本格 / 黄金年代书单（约 70 本），可勾选已读、增删书目、打题材标签，按分区 / 标签 / 作者三视图浏览。**分区可自定义**：添加书目时在分区下拉选「+ 新建分区…」即可创建新分区名，并在「数据」页重命名 / 删除。
- **拆解库**：记录每本书的诡计核心、线索布置、反转点、公平性自评、可借鉴点。**技法标签可自定义**：在拆解里输入新技法即加入标签池，并自动成为灵感矩阵的新行。
- **单本关系图**：节点-连线画出一本书内部的人物 / 线索 / 诡计关系。顶部工具栏：加节点 / 连线 / 改名 / 删除 / 完成 / 导出 PNG，拖动移动，关闭自动保存。
- **关系网**：把书做成节点，建立带类型和异同备忘的跨书关联。可在图上「＋连线模式」直接拖连两本，点已有连线即可编辑备注；也保留底部表单录入。可选显示共享标签的结构虚线。支持导出 PNG。
- **灵感矩阵**：技法 × 题材 的交叉表，空格 = 没人组合过的诡计，点一下即生成一张灵感卡。支持导出 PNG。
- **偏食分析**：标签 / 作者分布与技法盲区。
- **灵感箱**：随手记原创诡计点子，挂到启发它的书上。
- **对照**：并排比较两本书的拆解，一键存为关系网关联。
- **数据**：一键导出 / 导入 JSON 备份（含全部数据），或导出 Markdown 素材库。

## 使用与备份

数据存在浏览器 localStorage。为避免被浏览器清掉：

- 在「数据」页定期**导出 JSON 备份**；
- 换新版本文件时，先在旧文件导出、再到新文件导入；
- 想要更稳定的存储与图谱加载，在文件目录运行本地服务器：

```bash
python -m http.server 8000
# 访问 http://localhost:8000/
```

> 「单本关系图」「关系网」用到 CDN 上的 [vis-network](https://visjs.github.io/vis-network/)，首次需联网，之后浏览器缓存。其余功能完全离线。

## 部署到 GitHub Pages（可选）

把这些文件推到仓库，`index.html` 作为入口，在 Settings → Pages 选择分支即可。

## 贡献书单

只需编辑 `books.js`，每条格式：

```js
[分区号, "书名", "作者", "一句话概括", 是否默认已读(1/0)]
```

分区号 0–9 的含义见 `books.js` 顶部注释。
